Deep Dive

Why GitHub Actions Keeps Going Down

GitHub's ongoing Azure migration is creating systemic reliability challenges.

Azure Migration Pains

GitHub has been migrating its core platform from its legacy data center to Microsoft Azure. Actions and Copilot were migrated in 2024, but the core platform only began migration in October 2025. Traffic is currently split between the legacy data center and Azure, creating additional complexity and failure surface area.

Cascading Failures

GitHub's tightly coupled architecture means that Actions outages frequently cascade into other services. The Feb 2 runner outage also took down Copilot, CodeQL, Dependabot, GitHub Enterprise Importer, and Pages — all sharing the same compute infrastructure.

AI Agent Impact

Modern AI coding agents (Claude Code, Codex CLI, Cursor) depend on GitHub for commits, PRs, and CI/CD pipelines. When GitHub goes down, entire automated development workflows halt — even though agents can still generate code locally.

Every Service Is Struggling

Per third-party tracking, Actions is at 96.55% uptime, Webhooks at 96.79%, Pull Requests at 96.88%, and even Git Operations — the most basic function — is at 97.42%. No GitHub service is above 98% right now.
